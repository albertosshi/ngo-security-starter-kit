# Password and MFA Checklist

Use this checklist to reduce the risk of account takeover.

## Multi-factor authentication

- [ ] MFA is enabled for all email accounts.
- [ ] MFA is enabled for admin accounts.
- [ ] MFA is enabled for password managers.
- [ ] MFA is enabled for cloud storage accounts.
- [ ] MFA is enabled for finance, donation, and payment platforms.
- [ ] At least two trusted people can recover critical admin accounts.
- [ ] Recovery methods are documented and kept secure.

## Passwords

- [ ] Passwords are not shared in email, Slack, WhatsApp, or documents.
- [ ] The organization uses a password manager.
- [ ] Every important account has a unique password.
- [ ] Shared accounts are avoided where possible.
- [ ] Default passwords have been changed.
- [ ] Old passwords are changed when someone leaves the organization.

## Account recovery

- [ ] Recovery email addresses are still valid.
- [ ] Recovery phone numbers are still valid.
- [ ] Backup codes are stored securely.
- [ ] Admin accounts are not tied to a single person only.

## Red flags

Investigate immediately if:

- someone receives unexpected MFA prompts
- an email account sends messages the owner did not write
- login alerts appear from unknown locations
- password reset emails arrive unexpectedly
- files are shared publicly without explanation
